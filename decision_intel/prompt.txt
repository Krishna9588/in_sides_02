## What was built

**`model_connect.py`** — the single LLM gateway. Change `DEFAULT_PROVIDER` and `DEFAULT_MODEL` at the top and the entire project switches models. Supports Gemini, Claude, and OpenAI. Has a `model_connect_json()` convenience wrapper that auto-parses JSON back to a dict.

**`agent1_orchestrator.py`** — coordinates all three Agent 1 data sources (A: competitor profile, B: scrapers, C: internal files). Creates the project document in MongoDB (or falls back to `data/projects/groww.json` if Mongo is unavailable). Runs steps in sequence and saves status after each.

**`agent2/3/4.py`** — each loads the project doc, calls the LLM via `model_connect`, and writes its output back to the same project doc. The flow is: problems → insights → product briefs.

**`pipeline.py`** — single entry point to run everything:
```bash
python pipeline.py --project Groww --domain groww.in
python pipeline.py --project Groww --start-from agent2   # resume after agent1
python pipeline.py --project Groww --only agent3          # re-run just one step
```

## Three things to hook up before it runs

1. **Scraper function names** — in `agent1_orchestrator.py`, the imports like `from reddit_6_working_f import scrape_reddit` assume a function called `scrape_reddit`. Check the actual function names in your scraper files and update those 4 import lines.

2. **`.env` file** — copy `.env.example` to `.env`, add your `GEMINI_API_KEY` (or `ANTHROPIC_API_KEY`), and your `HF_TOKEN`.

3. **MongoDB** — if you have Atlas, paste the URI into `.env`. If not, it silently falls back to local JSON files — works fine for now.

## What to build next (in order)

1. **`rag_ingest.py`** — after pipeline completes, embed all the project data (problems, insights, briefs, signals) into Supabase pgvector. This is the RAG layer.
2. **`chat.py` (Agent 5 / Copilot)** — takes a natural language question, retrieves relevant chunks from Supabase, passes them to the LLM, returns a cited answer.
3. **FastAPI backend** — wraps `pipeline.py` and `chat.py` as REST endpoints so the frontend can call them.
4. **React dashboard** — project list, pipeline status, problems/insights/briefs viewer, and the chat box.

**No logos, no ASCII art** — just clean `[HH:MM:SS] [PASS] test_name` lines.

**Every test function is independently callable** — the runner uses `inspect.signature` to detect whether to pass `tmp_dir`, so you can call any test function directly from another file if needed.

**Two tiers of tests:**

| Tier | What it tests | Needs API key? |
|---|---|---|
| Default (fast) | Imports, structure, prompt building, DB fallback, platform detection | No |
| `--live` | Real LLM calls for agent2/3/4, full end-to-end pipeline | Yes |
